Data Visualisation
Proof of Concept

This page provides links to all materials provided in the Data Visualisation Proof of Concept (DataVis PoC) project delivered to Recycling Victoria (RV) when part of the Victorian Department of Energy, Environment and Climate Action (DEECA), in May 2026.

It includes links to: all completed data visualisation concepts and associated documentation (to guide RV's use of the visualisations and provide findings and explanation in the form of data + design notes); and summary material provided and presented as part of the project engagement.

In short, this page (and code repository) serves as a comprehensive compilation of all project materials.

Project documentation

Links to materials used to develop and deliver of this project:

  • Project code repository: Github code repository. This repository is set to public to facilitate code sharing (as a deliverable) to RV.
  • User guide: A brief set of guidance notes for exporting the visualisations as static files; and for embedding in websites.
  • Project data + design notes: Summary of delivered design concepts (i.e. 'recommendations'), including findings and insights from the data and design process.

  • Data tables used: Example spreadsheet with copies of the publicly available datasets from the Victorian Waste Performance and Projection Model and Victorian Local Government Waste Data release notes.

    An additional table is used to record data on Circular Economy metrics extracted manually from the Circular Economy Market Report. The usage of this table will only be refined and documented in a production version of these visuals.

    All tables are exported to text files (.tsv) and included with the source code (see user guide).

Data Visualisation

Three visualisation concepts were developed to a fully working, proof of concept stage. Each is delivered as a 'digital first' graphic that supports screen responsiveness and (better) accessibility and more flexible usage. All visuals are rendered as vector graphics and can be exported as such. See user guide and the data + design notes for more information.

All visualisations can be rendered without the sidebar/header section in situations where RV may want to use the graphics with custom headers and annotation (e.g. in presentation slides). These 'visOnly' links are provided for each visualisation below.

All visualistions default to the latest year of available data. This can be overridden for a given year of 20XX using a url query parameter of "year=20XX". This may be useful if visualisations are embedded in a website that should show a specific year (e.g. as part of a page that reports on a specific year). Examples are provide below (as observation of the url will better explain how to use the year query parameter).

1. The System

A representation of the "Shape of the Circular Economy" that focuses on the waste management flows (i.e. where most detailed data is available). This visualisation is available in four layout modes that RV are free to choose from and use for different use cases. Illustration components for this visual were created for the PoC for RV to use at their discretion: these are provided in separate links.

  • Mode d: The most stylised and dramatically angled version of the graphic that resembles a "d" shape. Available with illustration that includes an extra section for preserved (natural) resources. Examples with with visOnly and with year fixed to 2024.
  • Mode T: The most conventional version of the graphic that resembles a "T" shape. Available with illustration. Examples with with visOnly and with year fixed to 2023.
  • Mode b: A version of the graphic where the linear section is "b"(alanced) on an angle that is determined by the recovery vs landfill rates. Available with illustration. Examples with with visOnly and with year fixed to 2022 .
  • Mode 6: An version of the graphic with the extreme angle and reversal of linear flows to imply that linear economy is 'going backward' while the 'circular economy' moves forward. Available with illustration (without preserved resources). Examples with with visOnly and with year fixed to 2021. Note: This is not suggested as there are legibility/text direction issues that were considered to detract too much from the design intent.

2. The Journey

A representation of where Victoria has come from and where we are now, with regard to the Circular Economy. This data used in this graphic has been curated to focus heavily on the shapes of changes and trends in recovered waste.

  • Trends & Change: The proposed graphic composed of two sections that: 1) uses Joy Plots (aka ridgeline plots) to show and compare the percentage change of waste generated and recovered, for all waste source sectors and material streams; 2) uses a sereis of small multiple area sparkline plots to compare the trends in recovery rates for all waste source sectors and material streams.

    Accompanying this graphic (in the sidebar) are a set of commentary/annotation notes to help the audience connect the patterns they observe, with events and initiatives.

3. The Targets

A data graphic that captures Victoria's progress (since 2020) in achieving the four waste targets.

  • Target fans: Proposed data graphic featuring a variant on a sunburst chart to represent progress towards the 2030 targets. In this version, the progress tracks outward from the center.

X. Other stuff

Anything else worth sharing is put here:

  • Verspil De Stijl: A bonus 'for fun' graphic of waste breakdown by stream and source, in the style of Piet Mondrian. Default colours are adapted to use the EPA secondary colour pallette. A primary EPS blues palette is also available.

  • Journey drafting pieces: An earlier version of the Journey plot that considered the possible data (timeline series) that might tell parts of the data story. A conclusion from this exploration was the need to curate and distill the data.

  • Towards to target: An alternative version of the Target fans graphic was explored where progress was shown to move toward the center 'bullseye' of the graphic. While elegant in theory, it was concluded that this inversion was not as intuitive as first thought, and caused some typographic issues that were not worth pursuing a solution for.
